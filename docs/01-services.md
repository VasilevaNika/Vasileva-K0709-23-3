# Сервисы

Логическое разбиение системы. На ранних этапах всё может жить в одном backend-процессе с отдельными модулями.

## Bot (Telegram)

- Приём команд и сообщений от пользователя в Telegram.
- Отправка карточек анкет, кнопок «лайк / пас», уведомлений о мэтче.
- Вызов API backend (HTTP): не хранит бизнес-данные, только токен и настройки бота.

## User

- Регистрация и учёт пользователя по `telegram_id` (после `/start`).
- Сессия/состояние диалога можно держать в памяти бота, FSM или в Redis.
- Поля: активность, дата регистрации.

## Profile

- CRUD анкеты: имя, возраст (или дата рождения), пол, город, описание, интересы.
- Загрузка и привязка фото (файлы — в S3/MinIO, в БД — ключ или URL).
- Расчёт или хранение **полноты анкеты** (для первичного рейтинга).

## Ranking

- **Первичный рейтинг:** возраст, пол, интересы, гео; полнота анкеты; число фото; соответствие **предпочтениям** просматривающего (возраст, пол, город).
- **Поведенческий:** лайки, соотношение лайк/пас, мэтчи, активность в чате (в т.ч. время суток).
- **Комбинированный:** взвешенная сумма первичного и поведенческого; опционально рефералы.
- Предрасчёт в таблице `profile_ratings` и пересчёт через Celery (по заданию — доп. баллы).

## Interaction

- Выдача ленты кандидатов (с учётом рейтинга и фильтров).
- Запись действий: лайк / пас (`swipes`), проверка взаимности, создание `matches`.
- Публикация событий в очередь (опционально) для аналитики и рейтинга.

## Chat

- Сообщения внутри мэтча: кто написал, текст, время (`messages`).
- Для поведенческого рейтинга: частота сообщений, кто инициирует диалог после мэтча.

## Инфраструктура (не отдельные «сервисы» в коде, но узлы системы)

| Компонент | Роль |
|-----------|------|
| **PostgreSQL** | Основные данные: пользователи, анкеты, свайпы, мэтчи, сообщения, рейтинги. |
| **Redis** | Кэш пачки отранжированных анкет на сессию (этап 3 / доп. баллы). |
| **Celery** | Отложенный пересчёт рейтингов и фоновые задачи. |
| **RabbitMQ / Kafka** | Поток событий (лайк, просмотр, сообщение) между частями системы. |
| **MinIO (S3)** | Хранение файлов фотографий. |
